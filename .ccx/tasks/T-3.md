# M9: commit message hygiene — style mirror + marker strip

## Goal

Independent of T-2 — this touches only the loop's commit step, not
state location. It depends on T-1 because it consumes the shared config
resolver for `ccx.dogfood` and `ccx.commit.trailer`.

Pipeline (in `/ccx:loop` and `/ccx:forever`, run before each commit):

1. Worker produces draft commit message (as today).
2. **Style-mirror pass.** Feed
   `git log --pretty='%s%n%b%n--' -30 <integration-branch>` + draft
   message to a single LLM rewrite call. Prompt asks: "Rewrite the
   proposed commit message to match the repo's existing convention
   (prefix style, subject case, imperative vs. past tense, trailing
   period, body presence). Strip any task IDs (T-NN) or tooling
   markers. Output the rewritten message only."
3. **Marker-strip regex gate.** Match tooling-marker shapes, not
   ordinary product words:
   `(?i)(^|\s)(T-[0-9]+:|\[T-[0-9]+\]|\bT-[0-9]+\b|supervisor:\s*(dispatch|update board)?|ccx/)`
   against subject + body. If hit, regenerate (go back to step 2).
   Three consecutive hits → abort worker with `exit_status:
   commit-marker-leak` (new status). Supervisor picks this up via
   M5's stuck-exit auto-revise.
4. **Optional opt-in trailer.** If `ccx.commit.trailer = true`,
   append `Ccx-Task: T-X` Git trailer to the body (parseable by
   `git interpret-trailers --parse`). Default `false`.

Dogfood mode (`ccx.dogfood = true`):
- Skip steps 2 and 3 entirely. Worker's draft message lands as-is
  (`T-X:` prefix, `supervisor:` prefix all allowed).
- Trailer flag still respected.

`docs/supervisor-design.md` §18 (created by T-1) gets a subsection
documenting the rewrite prompt template and the regex.

## Acceptance

- [ ] Customer-mode `/ccx:loop` run on a task whose natural message
      would contain `T-N` lands a commit whose subject and body
      match the regex zero times.
- [ ] Three consecutive regex hits trigger
      `exit_status: commit-marker-leak`, surfaced to supervisor.
- [ ] `ccx.commit.trailer = true` appends a parseable Git trailer.
      Default false produces no trailer.
- [ ] Dogfood mode passes the draft through unchanged.
- [ ] `docs/supervisor-design.md` §18 documents the prompt and
      the regex.
- [ ] Changes limited to paths matching `scope.include`.
- [ ] Codex review returns `verdict: "approve"` with zero in-scope
      findings at the worker's `--min-severity`.

## Context

Source: `BOARD.md`. Project direction at dispatch time:

> Customer-mode invariant 3 — no commit subject or body on worker branches or new integration commits contains ccx tooling markers: task IDs in marker form, `supervisor:` subjects, `supervisor: dispatch`, `supervisor: update board`, or `ccx/` branch/path markers. Single exception: opt-in `Ccx-Task: T-X` trailer when `ccx.commit.trailer = true` (default false). T-3 enforces this at *write* time inside the worker. T-6 enforces it again at *merge* time as the final gate — two layers because the worker's LLM may regress and one-layer regex is fragile.

## Test plan

- Stage a synthetic commit where the worker's draft is
  `T-5: add CSV export to reports` on a repo whose convention is
  conventional commits (`feat: ...`). Verify the landed commit
  subject is something like `feat: add CSV export to reports` —
  the prefix is mirrored AND the `T-5:` is stripped.
- Inject a worker whose regenerated message still contains
  `supervisor: dispatch` three times in a row. Confirm worker exits

- Set `ccx.dogfood = true`, draft message `T-7: foo` — confirm it
  lands as-is.

## Decisions

- q: Use which LLM model for the style-mirror rewrite call?
  a: Match the worker's current implementer tier (M7 ladder rung). Don't introduce a separate model dimension. If the worker is on haiku, rewrite on haiku — message rewriting is well within haiku's competence.
- q: How many regex retries before declaring `commit-marker-leak`?
  a: 3. Matches M5's existing retry budget for analogous failures.
- q: Should the regex be configurable per repo (e.g., a user wants to allow `dispatch:` in their own messages)?
  a: No — keep the regex fixed in M9, but narrow it to tooling-marker shapes so ordinary product commits like "dispatch events" pass. Configurable regex is a rabbit hole; defer.
- q: When the marker-strip regex matches, do we rewrite via LLM again or attempt a naive in-line strip first?
  a: LLM rewrite. Naive strip leaves dangling syntax (`: subject` after `T-3:` is gone) and produces unnatural results. The whole point of style mirror is naturalness.

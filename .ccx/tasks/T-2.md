# M9: external worktrees — git worktree add under $STATE/worktrees/

## Goal

Move worker worktree spawn from the supervisor's current location
(sibling of the repo: `~/Repositories/ccx-loop-T-X`) into
`$STATE/worktrees/<task-key>/`, where `$STATE` is resolved by the
helper introduced in T-1.

Defaults:
- `<task-key>` = `T-X` (readable in `$STATE` introspection).
- `.git/worktrees/<task-key>/` metadata directory keeps the same
  readable name.

Paranoid mode (`ccx.paranoid = true` in the per-repo config):
- `<task-key>` = first 8 chars of `SHA-256(T-X || timestamp)`.
- `.git/worktrees/<hash>/` also opaque.
- Mapping `T-X ↔ hash` recorded in `$STATE/worktrees/_index.json`.

Wire-up:
- The supervisor's worktree-add invocation (the Phase A step that
  creates the worker worktree) computes the target path from T-1's
  helper + `worktrees/<task-key>` and passes it to
  `git worktree add`.
- On worker finish (any `exit_status`), the supervisor runs
  `git worktree remove --force $STATE/worktrees/<task-key>` *before*
  the branch delete in T-4 fires.

Dogfood mode (`ccx.dogfood = true`):
- Worktrees stay at the legacy sibling-of-repo path
  (`<repo-parent>/<repo-basename>-T-X`). No change for this repo.

`docs/supervisor-design.md` §18 (created in T-1) gains a worktree
relocation subsection.

## Decisions

- q: Should the worker's `cwd` inside the worktree be set explicitly, or rely on `git worktree add` and `cd`?
  a: Explicit `cd "$STATE/worktrees/<task-key>"` before the worker's `claude -p` invocation. Don't rely on shell side-effects.
- q: What if the user's repo lives on a different filesystem than `$XDG_DATA_HOME` and crossing FS boundaries breaks anything in git's worktree machinery?
  a: It doesn't — git worktrees support cross-filesystem locations natively. If a user reports a corner case, that's a follow-up; don't preempt.
- q: `_index.json` for paranoid mode — append-only or full rewrite each time?
  a: Full rewrite. The file is small (one entry per worker), worker count is bounded, and atomic rewrite avoids partial-update consistency issues.
